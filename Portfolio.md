## 项目概述

**时光印记** — 一款面向年轻人的旧时光寻人工具型 App。用户翻出老照片或旧信件，AI 自动修复和提取线索，生成寻人海报，一键发布到社交媒体，App 内沉淀数据支持 AI 自动匹配同线索用户。

**关键词**：AI 图像修复 · OCR 文字识别 · 智能匹配 · 社交传播 · 情感化设计

**我的角色**：独立完成全流程 — 产品定义、用户研究、竞品分析、交互设计、视觉设计、前后端全栈开发

---

## 产品简介（一句话版）

把模糊的记忆变成可传播的线索 — AI 帮你整理、海报帮你扩散、匹配帮你找到。

## 产品简介（详细版）

市面上不缺寻人平台，但都太沉重了 — 打拐、走失老人、公益审核，年轻人不会觉得自己需要。时光印记切的是"旧友重逢"这个没人做好的赛道：你翻到一张小学毕业照、一封旧信、一个模糊的名字，AI 帮你修复照片、提取线索、生成海报，你分享到朋友圈或小红书，然后等人找上门。App 内还有自动匹配引擎，对方也在找你的话，系统会通知你们。

---

## 解决的问题

| 用户痛点 | 解决方案 |
|---------|---------|
| 只有模糊线索（名字、学校、大概年份），不知道怎么找 | AI 从照片/信件中自动提取关键信息，补充用户描述，生成结构化线索 |
| 专门去发布寻人信息觉得"太隆重" | 轻量化流程，拍照→AI 处理→发布，三步完成 |
| 一个人找太无力，需要扩散 | 自动生成寻人海报，一键分享到微信/朋友圈/小红书 |
| 不知道对方是否也在找自己 | AI 引擎交叉比对所有寻人信息，命中时自动推送 |

## 产品功能

### 核心功能

- **老照片上传与 AI 修复** — 拍照/相册选择 → AI 画质修复和人脸增强 → 返回高清照片
- **信件上传与文字提取** — 拍摄旧信件 → OCR 识别手写/印刷文字 → AI 提取人名、地名、时间等关键信息
- **寻人线索描述** — 用户补充文字描述，支持纯文字模式跳过照片/信件上传
- **AI 生成寻人海报** — 修复后照片 + 关键信息 + 用户描述 → 多模板寻人海报
- **一键发布社交媒体** — 系统分享面板，支持微信/朋友圈/微博/小红书
- **AI 自动匹配** — 多维度交叉分析（时间、地点、人名相似度），匹配时推送通知
- **站内私信** — 匹配双方沟通通道
- **已找到标记与下架** — 从公共池移除，存入个人档案

## 技术架构

| 层级 | 技术选型 |
|------|---------|
| 前端 | React 19 + Vite 6 + TypeScript 5 + Tailwind CSS 4 |
| 后端 | Express 4 + TypeScript + SQLite (sql.js) |
| 认证 | JWT 手机号验证码 |
| AI 模型 | Real-ESRGAN + GFPGAN（图像修复）、PP-OCRv5（OCR）、DeepFace + ArcFace（人脸匹配） |
| 部署 | 前端 Cloudflare Pages + 后端 Railway |

### 架构亮点

- **全类型安全**：前后端共享 TypeScript 定义，编译时捕获类型错误
- **AI 功能全模拟 + 真实 API 预留**：开发阶段 AI 用 mock 数据，架构上预留了模型替换接口，上线时切换到真实模型无需重构
- **轻量数据层**：SQLite 单文件数据库 + sql.js 纯 JS 引擎，无外部数据库依赖，Demo 即可运行
- **手机壳模拟器**：桌面端开发时用 iPhone 尺寸包裹渲染，真机等比预览，适合录屏演示

## 面试 Q&A 准备

### Q：为什么做这个项目？

小时候在农村长大，邻居家的小孩搬走后，就再也没有联系了。后来发现身边很多人都有类似的经历——翻到旧照片会想起一个人，但不知道从哪里找起。市面上没有产品专门解决这个场景，就自己做了。

### Q：最大的技术挑战是什么？

发布流程是 6 步表单，步骤之间有关联状态（上传照片→AI 处理→展示结果→补充信息→海报编辑→发布）。如果用户只输入文字不传照片，需要跳过 AI 处理的两步。用自定义 Hook 管理多步骤状态 + 条件路由，保证状态在步骤间不丢失。

另一个是手机壳模拟器 — 桌面端开发时需要预览手机效果，要把所有页面包在一个 iPhone 框里。但页面里有大量 `fixed` 定位的元素（底部导航栏、FAB 按钮、分享菜单），`fixed` 是相对视口的，在手机框里就错位了。解决方案是 PhoneFrame 组件创建独立层叠上下文（`isolation: isolate`），把所有 `fixed` 改为 `absolute`，只有最外层容器保留 `fixed inset-0`。

### Q：AI 功能是真的还是 mock 的？

目前是 mock 数据 + 纯字符串匹配，但架构上预留了模型接口。效果演示用 mock 数据响应，上线时只需替换 API 调用层。

### Q：这个项目能商业化吗？

核心变现逻辑是订阅制。竞品分析发现类似工具（老照片修复 App）的付费转化率在 8-15%，而寻人场景的情感驱动力更强——愿意花几十块去找一个惦记了 20 年的人。

### Q：为什么用 SQLite 不用 PostgreSQL？

Demo 阶段 SQLite 零配置就能跑，展示时不需要启动外部数据库服务。生产环境替换为 PostgreSQL 只需要改数据库连接层代码。
